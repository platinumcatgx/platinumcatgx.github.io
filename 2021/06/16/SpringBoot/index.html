Spring Boot

为什么使用Spring Boot:

源于Spring的开发简单, 配置地狱问题, Spring Boot为Spring提供了一种快速启动的方式

Spring Boot并不是一项技术, 它并不是为了解决了什么问题, 它只是为Spring提供了一种快速使用的方式

解决了Spring的部分缺点

  1. 配置繁琐, 尤其是在整合第三方的技术时, XML配置会显得更加复杂
  2. 依赖繁琐, 在搭建环境时, 需要分析坐标库, 以及版本, 版本的兼容性又是会严重影响开发的进度

Spring Boot的怎么解决的?

  1. 自动配置

    是一个应用程序启动时的过程, 考虑了众多因素, 才决定应该用哪个, 该过程时springboot自动完成的

  2. 起步依赖

    本质上是一个maven项目对象模型, 定义了对其他库的依赖传递, 这些东西加载一起即支持某项功能;

    简单来说, 起步依赖就是将具备某些功能的坐标打包在一起, 并提供一些默认的功能

  3. 辅助功能

    例如, 嵌入式服务器, 安全, 指标, 健康检测, 外部配置等

为什么如此简洁?

spring boot 主要解决了两个问题: 依赖和配置

起步依赖: 在spring-boot-starter-parent,spring-boot-dependencies中固定了版本信息和依赖, 也就是说, 父工程定义了需要的大部分依赖和依赖版本, 我们在引入时只需要引入对应的依赖, 而不需要设置版本

Spring Boot的配置

spring boot是基于约定的, 很多配置都有默认值, 需要使用自己配置的话, 可以在application.yml/application.properties文件中进行配置, 配置文件使用的yaml/properties的语法, 都是比较简介的, 推荐使用YAML作为配置文件
